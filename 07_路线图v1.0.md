## 五、v1.0 闭环三大件

### 1. 回测校准(backtest.py 接真实数据)

- 用 T 年末可得数据评分 → 前 20% 等权 → 对比 T+1 同类均值,滚动 2019–2025
- 严格防前视:持仓类指标对齐披露滞后;禁用未来成立基金
- 产出:各维度权重的 RankIC 贡献 → **用证据校准权重**(替换当前先验)
- 专项检验:**长窗口 vs 短窗口谁对未来更有预测力**(呼应核心原则)

### 2. 3–6 个月滚动复盘机制 ★(核心原则的落地)

> 目的:体系不能"设定后就相信它",必须用客观事实持续自我校准。详见 [[research/基金评估-核心原则_过去与未来]]。

**设计**:

- **频率**:每 3 个月一次(季度),与基金季报披露节奏对齐
- **数据锚**:上一期(T-1)评分名单 vs 本期(T)实际表现
- **复盘四问**:
  1. 上期 Top 20%(候选/重点池)本期是否跑赢同类中位数?(命中率)
  2. 哪些规则误杀(被剔却表现好)/ 漏放(高分却暴雷)?
  3. 各维度得分与后续表现的相关性(RankIC)是否仍为正?哪个维度失效了?
  4. 是否有基金发生"历史可比性break"事件(换将/规模突变/风格漂移)而评分未及时反映?
- **产出**:季度复盘报告(存研究库)+ 权重调整建议 + 规则增删建议
- **实现**:`review.py` 脚本 + 季度定时任务;输入历年 archive/ 存档,输出复盘报告 md
- **闭环**:复盘结论反馈到 config.py 的权重/阈值,形成"评分→复盘→迭代"循环

**关键指标(复盘记分卡)**:
- 命中率:Top 组下期跑赢同类的比例(目标 >55%)
- RankIC:综合分 vs 下期收益的秩相关(目标 >0.05 且稳定为正)
- 规则有效性:各 N 规则剔除样本的下期表现(确认剔得对)
- 权重敏感性:维度权重 ±10% 扰动下名单稳定性

### 3. 深研流程跑通(L3)

- 用 `基金评估-深研模板` 对重点池 Top 5 做完整深研(经理画像+流程验证+归因深挖)
- 数据:且慢 Brinson/持仓 + Gangtise 调研纪要/研报 + 经理观点
- 验证模板可操作性,迭代模板

## 六、动态权重机制(贯穿 v0.3–v1.0,核心原则要求)

不止于静态记分卡,需引入:
- **历史可比性预警**:换将<2 年、规模突变、风格漂移近期发生 → 标记"历史分打折看",动态调低远期窗口权重
- **窗口权重可调**:5y/3y 权重不再固定 60/40,由回测证据 + 个体事件决定
- **市场范式感知**(探索性):某类历史能力是否因市场结构变化而贬值

## 七、里程碑顺序建议

1. v0.2 收尾(净值复权、剩余 bug、任期全量)— 1 个迭代
2. v0.3 C/D/E 维接入 — 重点,让评分名副其实
3. v1.0 复盘机制 review.py — 与 v0.3 可并行启动(用现有 A+B 评分先跑通复盘框架)
4. v0.4 固收线 — 可独立推进
5. v1.0 回测校准 + 深研 — 收口

> 复盘机制(五-2)是 v1.0 的灵魂,优先级仅次于维度补全。建议 v0.3 启动时同步搭 review.py 骨架。
