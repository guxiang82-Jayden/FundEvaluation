# 交接 · 债基 C 维数据接入(2026-06-14, Opus 4.8)

> 承 16 号(债基流水线串联)。本批用且慢 MCP 会话内拉取 C 维持仓数据,落 CSV + 加载层,使债基评分覆盖率从 0.75(ABDE)升满到 1.0(ABCDE)→ formal。同主动权益 cdim 模式。

## 一、产出

### 数据(scripts/data/)
- `cdim_bond_data.csv`:69 只代表性债基(按规模/人气跨子组选:纯债/短债/一二级/可转债/固收+/利率债/信用债/高杠杆),列:
  `credit_ratio, convertible_ratio, leverage_ratio, dur_sensitive, cr5_bond, neg_alert`
- `bond_universe_seed.csv`:种子名单(代码/名称/规模/成立日/近一年收益/类别),供扩样参考
- `bond_alert_codes.csv`:周跳水排雷名单(39 只,全市场口径;本批 69 只大基金均不在内)

### 代码(scripts/)
- `cdim_bond.py`(新):加载层。CSV → 派生 C 维评分输入:
  - `credit_sink` = 信用债净资产占比(C2,U型;中债隐含评级分布源在本部署为空,用此代理)
  - `duration_dev` = 敏感久期(C3,U型:距同类中枢近=稳)
  - `leverage_contrib` = 杠杆率(C4,越高套息贡献越大)
  - `leverage_ratio` / `neg_alert` 直通 screening_bond(FN4 杠杆超限 / FN5 踩雷)
  - C1 选股占比 selection_share_bond 由净值 Campisi 残差衍生,不在持仓层(待补)
- `run_monthly_bond.py`:已接入 `cdim_bond.load_cdim_bond`(L1 初筛前)
- `test_cdim_bond.py`(新):用真实 CSV 验证 C 维生效、覆盖率→1.0、formal
