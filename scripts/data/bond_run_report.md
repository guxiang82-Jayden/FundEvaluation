## 技术摘要

双 Track 已能完成真实取数、分类、初筛、Campisi、评分和 Excel 分表：

- 60 只请求中，58 只完成指标计算，2 只因东财返回 HTML 而失败。
- 初筛后 47 只进入标准通道，其中纯债 Track 38 只、固收+ Track 9 只。
- 固收+ 9 只均取得 `equity_position`，6 只稳健档可评分，3 只积极档因样本不足 5 只而 defer。
- 40/47（85.1%）标为 formal，4/47（8.5%）为 provisional，3/47（6.4%）未评分。

但当前结果**尚不适合直接作为正式榜单**。最重要的四个问题是：

1. 评分样本中存在 16 组重复份额，共 32/47 行，同一产品 A/B/C 份额互相参与分位并重复占榜。
2. 可转债基金进入纯债 `BOND` Track；小组回退后，可转债与短期纯债被合并进“纯债综合”比较。
3. FN3/FN4/FN5/FN6 数据覆盖严重不足，负面初筛实际上主要只有 FN1 规模规则生效。
4. 纯债 Track 的 Campisi R² 仍偏低：0 只达到 high，只有 9/38 达到 medium。

因此，本轮证明了“管道能跑”，尚未证明“分组与评分可信”。

## 8. 口径疑点与修正建议

### P0：评分前合并份额

债基 universe 应复用或重写稳健的份额合并规则，并合并规模。评分与 TopN 以底层基金为单位，输出时可附主份额代码和其他份额列表。

### P0：重新定义双 Track

当前纯债 Track 实际包含：

- 中长期纯债
- 短期纯债
- 混合债券一级
- 可转债基金

建议至少拆成：

1. 纯债：短期纯债、中长期纯债；
2. 一级债/普通债：混合债券一级；
3. 固收+：混合债券二级、偏债混合；
4. 可转债：单列专用记分卡，或至少归含权 Track。

### P0：禁止异质小组回退到同一个“纯债综合”

本批短期纯债 6 只、可转债 4 只都小于 10，最终一起进入“纯债综合”。短债与转债风险来源完全不同，跨组分位失真明显。

更稳妥的规则：

- 小组不足时 defer；
- 或只回退到预定义的相邻兼容组；
- 禁止可转债与纯债合并。

### P1：增加筛选数据覆盖率门槛

FN3-FN6 未充分取数时，应显式输出：

- `screening_data_coverage`
- `unchecked_rules`
- `screening_confidence`

不能把“字段为空而未触发”解释成“风险检查通过”。

### P1：Campisi 按子组重做因子和基准

- 中长期纯债可继续使用当前四因子，但需检查期限久期匹配。
- 短债应使用更短期限 level/slope 代理，不能与长债使用完全相同暴露解释。
- 一级债需考虑转债暴露。
- 可转债与固收+必须补转债/权益因子，或不展示可解释为择券能力的 alpha。

### P1：将“维度覆盖率”与“指标覆盖率”分开

formal 应至少要求关键指标齐全，而不是某个维度只出现一个指标就获得整维权重。特别是 C 维，应单独标记持仓数据是否命中。

### P2：明确 universe 政策

明确以下产品是“排除”还是“另建 Track”：

- 指数型-固收 673 只；
- QDII-纯债 60 只；
- QDII-混合债 23 只；
- 同业存单指数基金。

### P2：修复 Windows 测试路径

`test_bond_tracks.py` 应使用 `tempfile` 或项目临时目录，而不是写死 `/sessions/...`。本问题不影响真实 Excel 输出，但会造成 Windows 测试假失败。

### P2：真实调试改为分层抽样

`--limit 60` 当前按代码顺序截断，重复份额多且权益档分布不完整。建议调试模式按：

- 子组；
- 权益档；
- C 维是否命中；
- 基金成立年限

分层抽样，才能用较小样本覆盖全部路径。

## 9. 结论

本轮真实跑批验证了以下工程能力：

- 全市场 L0 分类可运行；
- 纯债/固收+ 双 Track 能自动分流；
- 固收+ 权益档和 defer 逻辑有效；
- 中债因子、Campisi、评分、Excel 分表均能落地。

但在修复份额合并、Track 定义、筛选覆盖率和 Campisi 子组适配前，输出应继续标记为**研发质检结果**，不应升级为正式债基推荐榜。
